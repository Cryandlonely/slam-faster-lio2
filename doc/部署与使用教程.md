## 1. 系统与 ROS 2 安装

### 1.1 安装 ROS 2 Humble

```bash
# 设置 locale
sudo apt update && sudo apt install -y locales
sudo locale-gen en_US en_US.UTF-8
sudo update-locale LC_ALL=en_US.UTF-8 LANG=en_US.UTF-8
export LANG=en_US.UTF-8

# 添加 ROS 2 APT 源
sudo apt install -y software-properties-common curl
sudo curl -sSL https://raw.githubusercontent.com/ros/rosdistro/master/ros.key \
    -o /usr/share/keyrings/ros-archive-keyring.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/ros-archive-keyring.gpg] \
    http://packages.ros.org/ros2/ubuntu $(. /etc/os-release && echo $UBUNTU_CODENAME) main" \
    | sudo tee /etc/apt/sources.list.d/ros2.list

# 安装 ROS 2 Humble Desktop
sudo apt update && sudo apt install -y ros-humble-desktop

# 自动 source（写入 ~/.bashrc）
echo "source /opt/ros/humble/setup.bash" >> ~/.bashrc
source ~/.bashrc
```

### 1.2 安装 colcon 构建工具

```bash
sudo apt install -y python3-colcon-common-extensions python3-rosdep
sudo rosdep init && rosdep update
```

## 3. Livox ROS Driver2 编译安装

工程中已内置 `src/livox_ros_driver2/` 子目录，**不需要单独克隆**，直接在工作区内构建即可。

```bash
cd ~/slam/slam-faster-lio2

# 初始化工作区（仅首次）
source /opt/ros/humble/setup.bash

# 单独先编译 livox_ros_driver2（其他包依赖它）
colcon build --packages-select livox_ros_driver2 \
    --cmake-args -DROS_EDITION=ROS2 -DHUMBLE_ROS=humble

source install/setup.bash
```

> ⚠️ 若遇到 `package.xml` 冲突，先手动执行：
> ```bash
> cd src/livox_ros_driver2
> cp package_ROS2.xml package.xml
> cp -rf launch_ROS2/ launch/
> cd ../..
> ```

## 5. 硬件与网络配置

### 5.1 Livox MID360 网络配置

MID360 通过以太网连接，需固定主机 IP：

```bash
# 将连接 MID360 的网口固定为 192.168.1.5（与雷达默认配置匹配）
# 建议在 NetworkManager 图形界面操作：
# 设置 → 网络 → 有线连接 → IPv4 → 手动
# IP: 192.168.1.5   子网掩码: 255.255.255.0   网关: 留空
```

或命令行临时设置（重启失效）：

```bash
sudo ip addr add 192.168.1.5/24 dev eth0   # eth0 替换为实际网口名
```

验证连通性：

```bash
ping 192.168.1.12   # MID360 默认 IP
```

雷达配置文件位置：[src/livox_ros_driver2/config/MID360_config.json](../src/livox_ros_driver2/config/MID360_config.json)

关键字段：

```json
{
  "host_net_info": {
    "cmd_data_ip":   "192.168.1.5",   // ← 主机 IP，与上面设置一致
    "point_data_ip": "192.168.1.5",
    "imu_data_ip":   "192.168.1.5"
  },
  "lidar_configs": [
    { "ip": "192.168.1.12" }          // ← 雷达 IP
  ]
}
```

### 5.2 底盘串口配置

底盘通过 USB-UART 连接，默认 `/dev/ttyUSB0`，115200 8N1。

```bash
# 查看串口设备
ls /dev/ttyUSB* /dev/ttyACM*

# 添加串口权限（一次性，重启后生效）
sudo usermod -aG dialout $USER
# 注销重新登录后生效
```

修改配置：[src/chassis/config/chassis_params.yaml](../src/chassis/config/chassis_params.yaml)

```yaml
chassis_node:
  ros__parameters:
    serial_port: "/dev/ttyUSB0"   # ← 按实际设备修改
```

### 5.3 RTK 串口配置

RTK 设备 USB 识别信息（VID=0x19D1，PID=0x0001，USB location 含 `.6`）已内置于 launch 文件，**插入后自动识别端口**，无需手动改配置。

若自动识别失败，手动指定：

```bash
# 查看 RTK 设备端口
python3 -c "
import serial.tools.list_ports
for p in serial.tools.list_ports.comports():
    print(p.device, hex(p.vid or 0), hex(p.pid or 0), p.location)
"
```

然后修改 [src/rtk/config/rtk_params.yaml](../src/rtk/config/rtk_params.yaml)：

```yaml
rtk_node:
  ros__parameters:
    serial_port: "/dev/ttyACM0"   # ← 手动指定
```

### 5.4 VRU（IMU）串口配置

修改 [src/vru_sensor/config/vru_params.yaml](../src/vru_sensor/config/vru_params.yaml)：

```yaml
vru_node:
  ros__parameters:
    port: "/dev/ttyS0"    # ← 按实际设备修改
    baud: 115200
```

| 配置文件 | 关键参数 | 说明 |

| [location/config/mid360.yaml](../src/location/config/mid360.yaml) | `acc_cov`, `gyr_cov`, `extrinsic_T/R` | FAST-LIO2 IMU 噪声和外参 |
| [nav_planner/config/slam_nav_params.yaml](../src/nav_planner/config/slam_nav_params.yaml) | `lookahead_distance`, `cte_kp`, `bypass_offset` | 路径跟踪和避障参数 |

| [rtk/config/rtk_params.yaml](../src/rtk/config/rtk_params.yaml) | `serial_port`, `reference.jinan_lat/lon` | RTK 串口和坐标参考点 |
| [bridge/config/bridge_params.yaml](../src/bridge/config/bridge_params.yaml) | `tcp_port`, `status_rate` | TCP 服務端口（默认 9090） |
| [vru_sensor/config/vru_params.yaml](../src/vru_sensor/config/vru_params.yaml) | `port`, `baud` | VRU IMU 串口 |
| [livox_ros_driver2/config/MID360_config.json](../src/livox_ros_driver2/config/MID360_config.json) | `host_net_info`, `lidar_configs.ip` | 雷达和主机 IP |

### 关键一致性约束

> ⚠️ 以下参数**必须在两处保持完全一致**，否则 RTK 坐标和导航坐标系会偏移！

```
rtk/config/rtk_params.yaml:
    reference.jinan_lat: 36.66111
    reference.jinan_lon: 117.01665

nav_planner/config/slam_nav_params.yaml:
    reference:
      latitude: 36.66111
      longitude: 117.01665
```

## 7. 启动流程（分步）

建议为每个节点开一个独立终端，先 source 环境：

```bash
source /opt/ros/humble/setup.bash
source ~/slam/slam-faster-lio2/install/setup.bash
```

### 场景 A：室内建图模式

```bash
# 终端 1 — Livox 雷达驱动
ros2 launch livox_ros_driver2 msg_MID360_launch.py

# 终端 2 — FAST-LIO2 建图（发布 /Odometry 和 /laser_cloud_world）
ros2 launch location mapping.launch.py config_file:=mid360.yaml rviz:=true

# 终端 3 — 底盘驱动（接收 /cmd_vel，发布 /chassis/feedback）
ros2 launch chassis chassis_launch.py

# 终端 4 — VRU IMU（可选，用于 EKF 预测）
ros2 launch vru_sensor vru_launch.py

# 终端 5 — Bridge TCP 服务（远程控制 + 状态推送）
ros2 launch bridge bridge_launch.py
```

建图完成后，保存地图（PCD 会自动实时保存到 `src/location/PCD/`）：

```bash
# 查看实时保存的地图
ls src/location/PCD/
```

### 场景 B：室外 GPS/RTK 导航模式

```bash
# 终端 1 — Livox 雷达驱动（RTK 模式下可选，用于避障）
ros2 launch livox_ros_driver2 msg_MID360_launch.py

# 终端 2 — 一键启动 RTK + 导航规划（含自动串口检测）
ros2 launch nav_planner slam_nav_launch.py

# 终端 3 — 底盘驱动
ros2 launch chassis chassis_launch.py

# 终端 4 — VRU IMU（提供高频速度反馈）
ros2 launch vru_sensor vru_launch.py

# 终端 5 — Bridge TCP 服务
ros2 launch bridge bridge_launch.py
```

### 场景 C：调试模式（自带 RViz）

```bash
# 一条命令启动 RTK + 导航 + RViz
ros2 launch nav_planner slam_nav_debug.launch.py
```

## 8. 日常使用场景

### 8.1 通过 TCP 远程控制（Bridge 接入）

TCP 端口：`9090`（控制），`9091`（PCD 传输）

**下发单点 GPS 目标：**

```bash
echo '{"cmd":"nav_goal","coord_mode":"gps","lat":36.66120,"lon":117.01680}' \
    | nc <机器IP> 9090
```

**下发多航点任务：**

```json
{
  "cmd": "nav_waypoints",
  "target_vel": 1.0,
  "waypoints": [
    {"lat": 36.66120, "lon": 117.01680},
    {"lat": 36.66130, "lon": 117.01700},
    {"lat": 36.66150, "lon": 117.01720}
  ]
}
```

**其他控制指令：**

| 命令 | 说明 |
|------|------|
| `{"cmd":"nav_cancel"}` | 取消当前导航 |
| `{"cmd":"nav_pause","value":true}` | 暂停（`false` 恢复） |
| `{"cmd":"start_mapping","config":"mid360.yaml"}` | 动态启动建图 |
| `{"cmd":"stop_mapping"}` | 停止建图并保存地图 |
| `{"cmd":"start_outdoor"}` | 切换到 RTK 定位源 |
| `{"cmd":"stop_outdoor"}` | 切换回室内 SLAM 定位 |
| `{"cmd":"query_status"}` | 查询当前状态（JSON） |

**状态推送格式（5 Hz）：**

```json
{
  "type": "pose",
  "state": "TRACKING",
  "lat": 36.661200,
  "lon": 117.016800,
  "yaw": 1.57,
  "dist_to_goal": 5.23,
  "cross_track_error": 0.12,
  "avoiding": false,
  "paused": false
}
```

### 8.2 通过 RViz 下发导航目标

1. 打开 RViz，选择 `2D Nav Goal` 工具
2. 在地图上点击目标点（发布到 `/goal_pose` 话题）

### 8.3 查看关键话题

```bash
# 当前位姿和状态
ros2 topic echo /nav_status
ros2 topic echo /current_pose

# 跟踪调试（CTE、航向误差、前视距离）
ros2 topic echo /tracking_debug

# 控制命令
ros2 topic echo /cmd_vel

# 底盘反馈
ros2 topic echo /chassis/feedback

# 点云（建图用）
ros2 topic echo /laser_cloud_world --no-arr
```

## 9. 常见问题

### Q1：`colcon build` 找不到 `livox_ros_driver2`

```
CMake Error: Could not find a package configuration file provided by "livox_ros_driver2"
```

**解决：** 先单独编译 livox 驱动并 source：

```bash
colcon build --packages-select livox_ros_driver2 \
    --cmake-args -DROS_EDITION=ROS2 -DHUMBLE_ROS=humble
source install/setup.bash
colcon build --cmake-args -DCMAKE_BUILD_TYPE=Release
```

### Q2：底盘串口权限被拒绝

```
[ERROR] chassis_node: Failed to open serial port /dev/ttyUSB0: Permission denied
```

**解决：**

```bash
sudo usermod -aG dialout $USER
# 注销重新登录，或临时：
sudo chmod 666 /dev/ttyUSB0
```

### Q3：RTK 串口自动检测失败

日志出现 `[RTK 串口检测] 未找到匹配设备`。

**解决：**

```bash
# 查找 RTK 设备实际端口
python3 -c "
import serial.tools.list_ports
for p in serial.tools.list_ports.comports():
    vid = hex(p.vid) if p.vid else 'None'
    print(f'{p.device}  VID={vid}  loc={p.location}')
"
# 然后直接在 launch 命令行覆盖（无需修改配置文件）
ros2 launch rtk rtk_launch.py serial_port:=/dev/ttyACM0
```

### Q4：FAST-LIO2 点云配准发散（定位漂移）

**排查顺序：**

1. 检查外参 `extrinsic_T` 是否准确（雷达到 IMU 的平移，mm 精度）
2. 下调 `acc_cov` / `gyr_cov`（更信任 IMU）
3. 增大 `max_iteration`（默认 5，可改 8）
4. 确认 `filter_size_surf` 不要过大（建图环境复杂时改 0.2）

### Q5：导航走 S 弯

调整 [slam_nav_params.yaml](../src/nav_planner/config/slam_nav_params.yaml)：

```yaml
tracking:
  cte_kp: 0.35            # 调小（原 0.8），减少横向纠偏幅度
  cte_dead_zone: 0.65     # 调大，吸收 GPS 抖动死区
  heading_kp: 0.5         # 调小，防止小误差快速打头
  cmd_filter_alpha: 0.12  # 调小，输出更平滑
  max_angular_velocity: 0.8
```

### Q6：避障触发后车撞上障碍

**可能原因：**

- `z_min/z_max` 范围不匹配雷达安装高度  
  → 雷达装在车顶 h 米高，则 `z_min = -(h-0.2)` 即地面以上 0.2m，`z_max = 1.5`
- `bypass_forward_clear` 太小，绕障点不够远  
  → 改大到 2.0–3.0m
- `hold_clear_frames` 太小，障碍还在视野边缘就解除  
  → 改大到 8–10

### Q7：地图文件路径错误

`location/config/mid360.yaml` 中的 `map_file_path` 是绝对路径，移植后需修改：

```yaml
/global_map_publisher:
    ros__parameters:
        map_file_path: "/home/<用户名>/slam/slam-faster-lio2/src/location/PCD/scans.pcd"
```

同样在 `/**:` 段下也有一个，两处均需修改。
