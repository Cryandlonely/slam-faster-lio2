## 1. 系统与 ROS 2 安装

### 1.1 安装 ROS 2 Humble

```bash
# 设置 locale
sudo apt update && sudo apt install -y locales
sudo locale-gen en_US en_US.UTF-8
sudo update-locale LC_ALL=en_US.UTF-8 LANG=en_US.UTF-8
export LANG=en_US.UTF-8

# 添加 ROS 2 APT 源
sudo apt install -y software-properties-common curl
sudo curl -sSL https://raw.githubusercontent.com/ros/rosdistro/master/ros.key \
    -o /usr/share/keyrings/ros-archive-keyring.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/ros-archive-keyring.gpg] \
    http://packages.ros.org/ros2/ubuntu $(. /etc/os-release && echo $UBUNTU_CODENAME) main" \
    | sudo tee /etc/apt/sources.list.d/ros2.list

# 安装 ROS 2 Humble Desktop
sudo apt update && sudo apt install -y ros-humble-desktop

# 自动 source（写入 ~/.bashrc）
echo "source /opt/ros/humble/setup.bash" >> ~/.bashrc
source ~/.bashrc
```

### 1.2 安装 colcon 构建工具

```bash
sudo apt install -y python3-colcon-common-extensions python3-rosdep
sudo rosdep init && rosdep update
```

## 3. 工程编译

工程所有包均在 `src/` 目录下，包含：

| 包名 | 说明 |
|------|------|
| `airy_driver/rslidar_sdk` | 速腾 Airy 雷达驱动，发布 `/rslidar_points` 和 `/rslidar_imu_data` |
| `airy_driver/rslidar_msg` | 速腾自定义消息类型 |
| `fast_lio_robosense` | FAST-LIO2 建图节点，订阅 Airy 点云和 IMU |
| `chassis` | STM32 底盘驱动 |
| `vru_sensor` | VRU IMU 驱动 |
| `rtk` | RTK 串口读取 + 室外里程计发布 |
| `nav_planner` | 路径规划 + 轨迹跟踪节点 |
| `bridge` | TCP 服务桥接（遥控通信 + PCD 传输） |

### 3.1 完整编译（首次）

```bash
cd ~/slam/slam-faster-lio2
source /opt/ros/humble/setup.bash

colcon build --cmake-args -DCMAKE_BUILD_TYPE=Release

```

```bash
colcon build --packages-select fast_lio_robosense   # 只重新编译修改的包
source install/setup.bash
```

### 3.3 永久 source 工作区

```bash
echo "source ~/slam/slam-faster-lio2/install/setup.bash" >> ~/.bashrc

```

## 4. 硬件与网络配置

### 4.1 速腾 RoboSense Airy 雷达配置

Airy 通过 UDP 网络连接，默认监听端口 msop=6699、difop=7788、imu=6688（驱动直接绑定本机端口，无需额外网络配置）。

雷达驱动配置文件：[src/airy_driver/rslidar_sdk/config/config.yaml](../src/airy_driver/rslidar_sdk/config/config.yaml)

关键字段：

```yaml
common:
  msg_source: 1           # 1 = 在线雷达
  send_point_cloud_ros: true
lidar:
  - driver:
      lidar_type: RSAIRY
      msop_port: 6699
      difop_port: 7788
      imu_port: 6688
    ros:
      ros_send_point_cloud_topic: /rslidar_points   # FAST-LIO2 订阅此话题
      ros_send_imu_data_topic: /rslidar_imu_data    # FAST-LIO2 订阅此话题
```

### 4.2 底盘串口配置

底盘驱动通过串口与 STM32 通信，默认设备为 `/dev/ttyUSB0`。

配置文件：[src/chassis/config/chassis_params.yaml](../src/chassis/config/chassis_params.yaml)

```yaml
chassis_node:
  ros__parameters:
    serial_port: "/dev/ttyUSB0"   # 根据实际设备路径修改
    max_vx: 2.0
    max_vy: 2.0
    max_vz: 1.0
```

确认设备节点：

```bash
ls /dev/ttyUSB*
```

如需修改，编辑 `chassis_params.yaml` 中的 `serial_port` 字段。

---

### 4.3 RTK 串口配置

RTK 模块默认串口为 `/dev/ttyACM4`，波特率 115200。

配置文件：[src/rtk/config/rtk_params.yaml](../src/rtk/config/rtk_params.yaml)

关键字段：

```yaml
rtk_node:
  ros__parameters:
    serial_port: "/dev/ttyACM4"
    baud_rate: 115200
    antenna_heading_offset_deg: 0.0    # 天线安装偏角（度），需现场标定
    reference.jinan_lat: 36.66111      # 坐标参考点纬度（与 slam_nav_params.yaml 保持一致）
    reference.jinan_lon: 117.01665     # 坐标参考点经度（与 slam_nav_params.yaml 保持一致）
```

> **注意**：`reference.jinan_lat` / `reference.jinan_lon` 必须与 `slam_nav_params.yaml` 中的
> `reference.latitude` / `reference.longitude` 完全一致，否则 RTK 坐标与导航航点坐标系会偏移。

也可在 launch 时临时覆盖串口：

```bash
ros2 launch rtk rtk_launch.py serial_port:=/dev/ttyACM3
```

---

### 4.4 IMU/VRU 串口配置

VRU（惯性测量单元）默认串口为 `/dev/ttyS0`，波特率 115200。

配置文件：[src/vru_sensor/config/vru_params.yaml](../src/vru_sensor/config/vru_params.yaml)

```yaml
vru_node:
  ros__parameters:
    port: "/dev/ttyS0"
    baud: 115200
    frame_id: "vru"
    accel_range: 0   # 0=±2g
    gyro_range: 0    # 0=±250 dps
```

## 5. 配置文件速查

| 文件 | 作用 | 关键参数 |
|------|------|---------|
| `src/fastlio/.../config/robosenseAiry.yaml` | FAST-LIO2 建图 | `lid_topic`, `imu_topic`, `extrinsic_T`, `pcd_save.*` |
| `src/airy_driver/rslidar_sdk/config/config.yaml` | Airy 雷达驱动 | `lidar_type`, msop/difop/imu 端口, ROS 话题名 |
| `src/nav_planner/config/slam_nav_params.yaml` | 导航规划 | `odom_topic`, GPS 参考点, 跟踪参数 |
| `src/chassis/config/chassis_params.yaml` | 底盘串口 | `serial_port`, 速度限幅 |
| `src/rtk/config/rtk_params.yaml` | RTK 串口 + GPS 参考 | `serial_port`, `antenna_heading_offset_deg`, 参考经纬度 |
| `src/vru_sensor/config/vru_params.yaml` | VRU 串口 | `port`, `baud` |
| `src/bridge/config/bridge_params.yaml` | TCP 桥接 | `tcp_port`(9090), `pcd_tcp_port`(9091), 建图 launch 包名 |

### 5.1 实时 PCD 传输配置（FAST-LIO2）

在 `robosenseAiry.yaml` 的 `pcd_save` 节下：

```yaml
pcd_save:
  pcd_save_en: true
  realtime_trans_save_en: false        # true 开启实时传输
  realtime_trans_interval_sec: 1.0     # 写入间隔（秒）
  realtime_trans_rel_path: "PCD/transPCD/trans.pcd"  # 输出文件路径（相对包根目录）
  realtime_trans_voxel_size: 0.2       # 体素降采样（m），≤0 不降采样
```

## 6. 启动流程（分步）

### 6.1 环境准备

```bash
cd ~/slam/slam-faster-lio2
source install/setup.bash
```

### 6.2 分步启动

> 各步骤在独立终端中运行，或使用 tmux/screen 分屏。

**步骤 1 — 雷达驱动**

```bash
ros2 launch rslidar_sdk start.py
```

**步骤 2 — IMU/VRU 驱动**

```bash
ros2 launch vru_sensor vru.launch.py
```

**步骤 3 — 底盘驱动**

```bash
ros2 launch chassis chassis_launch.py
```

**步骤 4 — 导航规划（含 RTK）**

```bash
# 调试模式（可视化话题、详细日志）
ros2 launch nav_planner slam_nav_debug.launch.py

# 正式部署模式
ros2 launch nav_planner slam_nav_launch.py
```

**步骤 5 — TCP 桥接**

```bash
ros2 launch bridge bridge_launch.py
```

### 6.3 一键启动（推荐）

```bash
./script/start_all.sh         # 启动全部并自动监控重启
./script/start_all.sh stop    # 停止全部
```

> `start_all.sh` 对每个模块在退出后自动重启（最多 10 次），日志写入 `nav_logs/` 目录。

### 6.4 纯建图模式

仅启动雷达和 FAST-LIO2，不启动导航：

```bash
# 终端 1
ros2 launch rslidar_sdk start.py

# 终端 2
ros2 launch fast_lio_robosense mapping_robosense_airy.launch.py

# 建图完成后保存地图
ros2 service call /map_save std_srvs/srv/Trigger
```

保存的地图默认写入 `PCD/scans/` 目录（由 `map_file_path` 控制）。

## 7. 日常使用场景

### 7.1 TCP 远程控制接口

`bridge` 节点在 **9090** 端口监听控制命令（JSON 格式），在 **9091** 端口提供地图点云传输。

**支持的命令：**

| 命令 | 说明 | 示例负载 |
|------|------|---------|
| `nav_goal` | 单点导航（GPS 或本地坐标） | `{"cmd":"nav_goal","lat":36.661,"lon":117.016}` |
| `nav_waypoints` | 多航点导航 | `{"cmd":"nav_waypoints","waypoints":[{"lat":36.661,"lon":117.016}],"target_vel":0.5}` |
| `nav_cancel` | 取消导航 | `{"cmd":"nav_cancel"}` |
| `nav_pause` | 暂停导航 | `{"cmd":"nav_pause"}` |
| `nav_resume` | 恢复导航 | `{"cmd":"nav_resume"}` |
| `query_status` | 查询系统状态 | `{"cmd":"query_status"}` |
| `start_mapping` | 启动 FAST-LIO2 建图 | `{"cmd":"start_mapping"}` |
| `stop_mapping` | 停止建图并保存地图 | `{"cmd":"stop_mapping"}` |
| `start_outdoor` | 切换到 RTK 室外定位模式 | `{"cmd":"start_outdoor"}` |
| `stop_outdoor` | 切换回室内 SLAM 定位 | `{"cmd":"stop_outdoor"}` |

**`query_status` 响应格式（bridge 周期广播）：**

```json
{
  "type": "pose",
  "state": "IDLE",
  "lat": 36.661110000,
  "lon": 117.016650000,
  "yaw": 0.5,
  "battery": 25.6,
  "vx": 0.0,
  "vy": 0.0
}
```

`state` 取值：`IDLE`、`PLANNING`、`TRACKING`、`REACHED`、`ERROR`

### 7.2 常用话题监控

```bash
# 导航状态
ros2 topic echo /nav_status

# 当前位姿（本地坐标）
ros2 topic echo /current_pose

# FAST-LIO2 里程计
ros2 topic echo /Odometry

# 底盘反馈（实际速度）
ros2 topic echo /chassis/feedback

# 注册点云（世界坐标系）—— 只看元数据，不打印数组
ros2 topic echo /cloud_registered --no-arr
```

## 8. 常见问题

### Q1：雷达启动后无点云，FAST-LIO2 无输出

1. 检查 `rslidar_sdk` 驱动是否正常启动：`ros2 topic list` 中是否有 `/rslidar_points`。
2. 检查 Airy 雷达网络连接——雷达 IP 默认 `192.168.1.200`，主机需与雷达在同一子网段。
3. 检查 `config.yaml` 中端口是否与雷达固件一致（msop=6699, difop=7788, imu=6688）。

### Q2：底盘无响应

1. 确认串口设备存在：`ls /dev/ttyUSB*`。
2. 检查 `chassis_params.yaml` 中 `serial_port` 是否与实际设备一致。
3. 检查权限：`sudo chmod 666 /dev/ttyUSB0`，或将用户加入 `dialout` 组：`sudo usermod -aG dialout $USER`。

### Q3：调整 FAST-LIO2 外参（IMU ↔ 雷达）

在 `robosenseAiry.yaml` 的 `mapping` 节下修改：

```yaml
mapping:
  extrinsic_T: [0.0167, 0.0165, -0.0046]   # IMU 到雷达的平移（m），需实测标定
  extrinsic_R: [1., 0., 0.,
                0., -1., 0.,
                0., 0., -1.]                # 旋转矩阵（行优先）
```

### Q4：RTK 定位坐标与导航航点偏移

`rtk_params.yaml` 与 `slam_nav_params.yaml` 中的 GPS 参考原点必须完全一致：

- `rtk_params.yaml`：`reference.jinan_lat` / `reference.jinan_lon`
- `slam_nav_params.yaml`：`reference.latitude` / `reference.longitude`

两者任意一处修改时必须同步更新另一处。

### Q5：串口被 ModemManager 占用（/dev/ttyACM*）

```bash
sudo systemctl stop ModemManager
sudo systemctl disable ModemManager
```
